# Prompt 10: Case Study Documentation — QMN KDP Publishing Engine

*Use this prompt to document a book project as a QMN/MindQuest case study.*

---

## When to Use This Prompt

After:
- Book has been submitted to KDP
- Initial post-launch data is available (30 days post-publication)
- Elise is ready to document the project for QMN case study purposes

---

## Prompt (Copy and use with Claude)

```
I'm documenting a completed children's book project as a QMN / MindQuest case study.

This case study is for internal learning and potential future external sharing. It must be honest — no inflated results, no misleading claims about outcomes.

BOOK PROJECT DETAILS:
- Title: [TITLE]
- Date research started: [DATE]
- Date submitted to KDP: [DATE]
- Date published (if applicable): [DATE]
- Current status: [Published / Under KDP Review / Rejected]

PRODUCTION DATA:
- Niche score: [SCORE]/100
- Key niche insight that led to this book: [SUMMARY]
- Total production phases completed: [LIST]
- Approximate total time from research to submission: [TIME]
- Tools used: [Claude, Canva, illustrator name/type, etc.]
- AI role vs. human role summary: [BRIEF DESCRIPTION]

POST-LAUNCH DATA (fill in what is available):
- Units sold (30 days): [NUMBER — be honest; 0 is a real result]
- BSR observed: [RANGE]
- Reviews received: [COUNT] Average rating: [RATING]
- Any listing changes made after launch: [YES/NO — describe if yes]
- Any KDP issues or flags: [YES/NO — describe if yes]

Please help me draft a case study using this structure:
1. Project Overview
2. Research and Niche Selection (what was found, what score was given, why this niche was chosen)
3. Production Process (what phases were completed, timeline, AI vs. human roles)
4. Copyright and Policy (checks completed, AI disclosure status)
5. KDP Submission (listing strategy, any submission notes)
6. First 30 Days (honest sales and performance data)
7. Post-Launch Adjustments (any changes made and why)
8. Lessons Learned (what would be done differently)
9. QMN Brand Impact (how this book served the brand)

IMPORTANT CONSTRAINTS:
- Do not inflate any results. Report actual numbers.
- Be specific about what AI did and what human judgment did.
- Do not present this as a "how to get rich on KDP" story.
- Do not make predictions about future sales.
- The case study must be honest enough to be genuinely useful for learning.
- Mark the draft as "CASE STUDY DRAFT — NOT APPROVED FOR EXTERNAL SHARING" until Elise reviews it.
```

---

## After Getting the Output

1. Save in `case_studies/[book-id]/case_study_draft.md`
2. Update at 30 and 90 days with new data
3. Elise reviews and approves before any external sharing
